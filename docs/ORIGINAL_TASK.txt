Ты -- опытный разработчки на Python.
Проанализируй функционал сайтов в папках Web и WebVA.

Описание сайта:
Эта образовательный сайт, в котором заходят пользователи двух типов: ученики и учителя.
Учителя могут прикреплять к себе класс учеников следующим образом: учитель выбирает нужный
город, школу и класс. Система постоянно автоматически ищет учеников по фильтру: город, школа и класс, а 
затем прикрепляет найденных в БД пользователей к учителю. После прикрепления пользователя автоматически
приходит уведомление учителю о том, что "ученик {name} {surname} из {city}, школы {school}, класса {grade}
был прикпеплён к вам". Также ученику приходит уведомление "Вы были прикреплены к учителю {name} {surname}, из города {city}".
Также на сайте нужно реализовать плановый групповой звонок учителя с классом. После созвона запись должна сохранятся в
течении трёх дней, которую можно загрузить пользователю и которая видна только определённому классу, с которым был созвон.
Класс учеником может оставлять комментарии под видео, чтобы учителю было видно, что ученику было не понятно.
На сайте должен быть личный чат учитель-ученик, ученик-ученик.
Учитель может видеть все свои прикрепленные классы в виде такого графического представления на отдельной странице:

                     | -- 10 Б
     |-- школа 74 -- | 
     |               |          | Иван Пупкин
     |               | -- 7а -- | Артем Демидов
Я -- |                          | ....
     |
     |-- школа 76 -- ....

Для удобства, учитель может открыть отдельную ветку с городом, где он может посмотреть все классы в этом городе,
или например открыть только класс, в таком случае будет виден только список учеником.
При просмотре отдельного класса/школы/ветки с городами, содержащими школы и классы 
учителю должно быть видно кто находится в сети: зеленым цветом -- в сети, красным -- не в сети.

Сам ученик может видеть своих учителей, а также видеть, в сети ли они, или нет.
Также ученик может пользоваться AI и формулами

Правила:
1. Разбивай логику по отдельным файлам, чтобы получить микросервисную архитектуру.
2. Если библиотека предоставляет необходимый функционал, то нет необходимости использовать другую
3. Все созданные тобой .txt, .md файлы нужно отдель от программы, поместив их в папку
4. Избегай дублирования кода, путем использования функций, циклов, классов.
